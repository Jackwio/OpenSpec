# CLI 檢視指令 - 更改

## 新增要求

### 要求：儀表板顯示

系統應提供 `view` 顯示規格和變更的儀表板概述的命令。

#### 場景：基本儀表板顯示

- **何時** 使用者執行 `openspec view`
- **然後** 系統顯示格式化的儀表板，其中包含摘要、活動變更、已完成變更和規格等部分

#### 場景：沒有OpenSpec目錄

- **何時** 使用者執行 `openspec view` 在沒有OpenSpec的目錄中
- **然後** 系統顯示錯誤訊息“✗ 找不到 openspec 目錄”

### 要求：摘要部分

儀表板應顯示包含關鍵項目指標的摘要部分。

#### 場景：完整摘要展示

- **何時** 儀表板呈現規格和更改
- **那麼**系統顯示規格和要求的總數
- **AND** 顯示正在進行的活動變更的數量
- **AND** 顯示已完成的變更數量
- **AND** 顯示整體任務進度百分比

#### 場景：空項目摘要

- **何時** 不存在任何規格或更改
- **那麼** 摘要顯示所有指標的計數為零

### 要求：活動更改顯示

儀表板應透過視覺進度指示器顯示活動變化。

#### 場景：帶有進度條的主動更改

- **何時** 任務正在進行更改
- **然後** 系統顯示每個更改，更改名稱左對齊
- **和** 使用 Unicode 字元的視覺進度條
- **和** 右側的完成百分比

#### 場景：沒有主動更改

- **何時** 所有變更均已完成或不存在任何更改
- **那麼** 活動更改部分從顯示中省略

### 要求：已完成的變更顯示

儀表板應在單獨的部分列出已完成的變更。

#### 場景：已完成的變更列表

- **何時** 已完成變更（所有任務均已完成）
- **那麼** 系統會在專用部分中以複選標記指示器顯示它們

#### 場景：混合完成狀態

- **何時** 有些變更已完成，有些變更則處於活動狀態
- **然後**系統將它們分成適當的部分

### 要求：規格展示

儀表板應顯示按需求數量排序的規格。

#### 場景：包含計數的規格列表

- **何時** 專案中存在規範
- **那麼** 系統顯示按需求計數（降序）排序的規格以及計數標籤

#### 場景：有解析錯誤的規格

- **何時** 無法解析規範文件
- **那麼** 系統將其包含在 0 個需求計數中

### 需求：視覺格式

儀表板應使用一致的視覺格式以及顏色和符號。

#### 場景：顏色編碼

- **何時** 顯示儀表板元素
- **那麼**系統使用青色作為規格項目
- **和** 黃色表示主動更改
- **和** 綠色表示已完成的項目
- **和** 補充文字為深灰色

#### 場景：進度條渲染

- **何時** 顯示進度條
- **那麼** 系統使用填充塊 (█) 表示已完成的部分，使用淺色塊 (░) 表示剩餘部分

### 要求：錯誤處理

view 指令應該優雅地處理錯誤。

#### 場景：檔案系統錯誤

- **何時** 檔案系統操作失敗
- **然後** 系統繼續使用可用資料並忽略無法存取的項目

#### 場景：無效的資料結構

- **何時** 規格或更改的格式無效
- **那麼**系統會跳過無效項目並繼續渲染
