# CLI 存檔命令 - 更改

## 修改後的要求

### 要求：規格更新流程

在將變更移至存檔之前，該命令應將增量變更套用至主要規格以反映部署的實際情況。

#### 場景：應用增量更改

- **何時** 使用基於增量的規範歸檔變更
- **THEN** parse and apply delta changes as defined in openspec-conventions
- **並且**在應用之前驗證所有操作

#### 場景：驗證增量更改

- **何時** 處理增量更改
- **然後** 依照 openspec-conventions 中指定的方式執行驗證
- **並且** 如果驗證失敗，則顯示特定錯誤併中止

#### 場景：衝突偵測

- **何時** 應用增量會建立重複的需求標頭
- **然後** 中止並顯示衝突的錯誤訊息
- **並**建議手動解決

## 新增要求

### 要求：顯示輸出

該命令應提供有關增量操作的明確回饋。

#### 場景：顯示增量應用程式

- **何時**套用增量更改
- **然後** 顯示每個規格：
  - 增加的要求數量
  - 修改的要求數量
  - 刪除的要求數量
  - 重命名的需求數量
- **AND** 使用 openspec-conventions 中定義的標準輸出符號 (+ ~ - →)：
  ```
  Applying changes to specs/user-auth/spec.md:
    + 2 added
    ~ 3 modified
    - 1 removed
    → 1 renamed
  ```
